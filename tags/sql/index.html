获取最后插入的ID: LAST_INSERT_ID()

从名字可以看出，LAST_INSERT_ID即为最后插入的ID值，它有2种使用方法:
一是不带参数：LAST_INSERT_ID()，这种方法和AUTO_INCREMENT属性一起使用，当往带有‘AUTO_INCREMENT’属性字段的表中新增记录时，LAST_INSERT_ID()即返回该字段的值，大家可试下（我已经验证过）；
二是带有表达式：如上面介绍的LAST_INSERT_ID(value+1)，它返回的是表达式的值，即‘value+1’；

SELECT LAST_INSERT_ID();

mysqldump命令

mysqldump是mysql的备份工具, 我们常用他来批量导出数据库,如:
mysqldump -uroot -p -h127.0.0.1 database_name >> xx.sql
这条是常用的数据库批量导出语句, mysqldump还有下面一些常用的导出的参数配置:

--where按条件查找

mysqldump按条件查找的参数是 -where/-w，例子如：

mysqldump -uroot -p123456 dbname table_name -where “id>90” > ~/db.sql

--extended-insert每行是否都有insert语句和字段

mysqldump -uroot -p123456 cocksucker_online sea_item_effect —extended-insert=false -c —where=” created<=’2014-07-17 00:00:00’” > ~/web/db/test.sql

--skip-tz-utc解决导出时时区错误问题

mysqldump -uroot -p123456 cocksucker_online sea_item_effect —skip-tz-utc —where=” created<=’2014-07-17 00:00:00’” > ~/web/db/test.sql

--default-character-set指定编码语句

带上参数--default-character-set=utf8就指定默认为utf8,通常用来解决导入导出遇到中文乱码问题。例如：

mysqldump -uroot -p —default-character-set=utf8 dbname tablename > bak.sql
mysql -uroot -p —default-character-set=utf8 dbname < bak.sql

--ignore-table排除指定表导出

mysqldump -uroot -p123456 database --ignore-table=database.table1 --ignore-table=database.table2 > backup.sql

--lock-tables=0不锁表导出

压缩导出文件

mysqldump -uroot -p --add-drop-table db_name | gzip > /path/db_back_name.sql.gz

MySQL时间相关函数

MySQL时间相关的处理函数有如下：

  • date()
  • date_format()

TIMESTAMPDIFF() 计算时间间隔

语法如：TIMESTAMPDIFF(interval,datetime_expr1,datetime_expr2)

  • interval：可以是年、月、周、日：year|month|week|day
  • datetime_expr1：开始时间
  • datetime_expr2：结束时间

一般来说开始时间是小于结束时间的，如果开始时间大于结束时间则得到负数。
