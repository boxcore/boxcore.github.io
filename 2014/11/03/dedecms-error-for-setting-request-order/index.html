DedeCMS报错要求修改request_order配置

By chunze.huang

11月 3 2014 Updated:1月 28 2015

Contents

自己安装的DA面板添加了dedecms的程序后提醒没有用户组权限。需要说明下，da面板中如果你的php是以cgi模式运行的话就可能遇到此类的问题。因为你很有可能把权限修改成了777的权限， 那么就要把网站目录递归修改成755的权限，因为cgi模式是禁止777的php文件执行的。

设置完后提醒

(PHP 5.3 and above) Please set ‘request_order’ ini value to include C,G and P (recommended: ‘CGP’) in php.ini

这是由于在PHP最新的版本中增加了一个配置项目“request_order”，默认值为“GP”，这个存在一定的安全风险。这里我们建议用户将配置更改为“CGP”

可以在phpinfo中查看对应的php.ini配置目录（默认在：/usr/local/etc/php5/cgi/php.ini），找到下面选项：

request_order = “GP”

更改为

request_order = “CGP”

重启服务器后即可。
